## 1. Identificação de Necessidades dos Usuários e Requisitos de IHC
**Que dados coletar?**
- Dados sobre o Perfil dos Usuários:

  - Faixa etária e formação profissional
  - Tempo de atuação em monitoramento ambiental/combate a incêndios

- Dados sobre Necessidades e Expectativas:

  - Principais dificuldades enfrentadas no monitoramento atual
  - Funcionalidades consideradas essenciais
  - Informações críticas durante um alerta de incêndio
  - Preferências de visualização de dados (mapas, gráficos, tabelas)

- Dados sobre Usabilidade:

  - Facilidade de navegação no sistema
  - Clareza das informações apresentadas
  - Tempo de resposta
  - Eficácia dos alertas e notificações
  
- Dados sobre o Contexto de Uso:

  - Dispositivos utilizados (desktop, tablet, mobile)
  - Duração dos turnos de monitoramento
  - Situações de estresse ou pressão de tempo
  
- Dados sobre Requisitos Funcionais:

  - Tipos de relatórios necessários
  - Integração com outros sistemas
  - Histórico de dados a ser mantido

**De quem coletar?**
- Usuários Primários:

  - Analistas de monitoramento ambiental e coordenadores/supervisores (central de controle): principais usuários do sistema que recebem notificações, geram relatórios e tomam decisões estratégicas.

- Usuários Secundários:

  - Bombeiros florestais e outros profissionais de campo: podem precisar acessar dados técnicos ou geográficos; valorizam informações rápidas e confiáveis para tomada de decisões em situações críticas.


## 2. Aspectos Éticos
**Seu projeto deverá considerar aspectos éticos? Justifique usando os conceitos da aula.**

Sim, o projeto deve considerar aspectos éticos. Justificativa:

- 1. Consentimento Informado
Os participantes devem ser informados sobre: objetivos da pesquisa, uso dos dados, tempo estimado de participação, direito de desistir a qualquer momento.
- 2. Privacidade e Confidencialidade
Dados pessoais devem ser anonimizados ou pseudonimizados; informações sensíveis (desempenho individual, críticas ao trabalho) não devem ser associadas a nomes; armazenamento seguro dos dados (conforme LGPD).
- 3. Não Maleficência
A coleta de dados não deve causar desconforto, estresse ou prejuízo aos participantes; evitar perguntas invasivas ou que exponham deficiências dos participantes; respeitar horários de trabalho (não atrapalhar operações críticas).
- 4. Beneficência
Os resultados devem trazer melhorias reais para o trabalho dos participantes; compartilhar os resultados e melhorias implementadas com os participantes.
- 5. Equidade e Justiça
Incluir diferentes perfis de usuários (iniciantes e experientes, diferentes faixas etárias); garantir acessibilidade nos instrumentos de coleta; não excluir participantes por características pessoais irrelevantes.


## 3.​ Ferramentas de Coleta de Dados (três técnicas diferentes)
**TÉCNICA 1: Questionário Online**
- Nome do Instrumento: Questionário de Perfil e Necessidades dos Usuários
- Objetivo: Coletar dados quantitativos sobre perfil dos usuários, experiência com sistemas similares, necessidades prioritárias e expectativas em relação ao sistema de monitoramento.
- Como Aplicar:
  - Criar questionário no Google Forms
  - Realizar teste piloto com 3-5 usuários

- Aplicação:

  - Enviar link por e-mail institucional
  - Prazo de 2 semanas para resposta
  - Garantir anonimato das respostas

- Análise:

  - Exportar dados para Excel/CSV
  - Realizar análise estatística descritiva
  - Gerar relatório de resultados

- Instrumento - Link do Questionário:
[https://forms.google.com]

**TÉCNICA 2: Entrevista Semiestruturada**
- Nome do Instrumento: Roteiro de Entrevista em Profundidade sobre Experiência de Monitoramento
- Objetivo: Obter dados qualitativos aprofundados sobre experiências, necessidades não explicitadas, contexto real de trabalho, fluxos de tarefas e aspectos emocionais dos usuários.
- Como Aplicar:
  - Selecionar 5-10 participantes de diferentes perfis
  - Apresentar objetivos
  - Seguir roteiro
  - Fazer perguntas de aprofundamento
  - Anonimizar dados
  
- Instrumento: Roteiro de Entrevista

**TÉCNICA 3: Análise em Grupo**
- Nome do Instrumento: Sessão de Análise em Grupo
- Objetivo: Gerar discussão coletiva sobre necessidades, identificar consensos e divergências, estimular proposição criativa de funcionalidades e validar requisitos através da interação entre usuários.
- Como Aplicar:
  - Recrutar 6-12 participantes por grupo
  - Fazer 2-3 grupos
  - Iniciar discussão
  - Administrador dos grupos coleta as informações (por meio de áudio, vídeo ou anotação)
  - Encerramento 
  - Analisar consensos e divergências

- Instrumento: Roteiro do Grupo Focal
